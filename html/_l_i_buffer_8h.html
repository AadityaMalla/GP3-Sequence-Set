Detailed Description

Reads and writes length-indicated ZIP code data files.

Author
CSCI 331 Team 2
Date
03/13/2026

A length-indicated (LI) file stores every record — header and data — as a 2-byte big-endian uint16_t length prefix followed by the packed payload bytes.

File layout: [2-byte len][header payload] <- HeaderBuffer writes/reads this [2-byte len][record payload] <- data record 1 [2-byte len][record payload] <- data record 2 ...

Data record payload (packed binary): [4-byte int zipCode] [c-string placeName\0] [c-string state\0] [c-string county\0] [8-byte double latitude] [8-byte double longitude]

Note
LIBuffer owns the file streams. HeaderBuffer and DataFileHeader are used as helpers.
