A length-\/indicated (LI) file stores every record — header and data — as a 2-\/byte big-\/endian uint16\+\_\+t length prefix followed by the packed payload bytes.

File layout:\+ \+[2-\/byte len]\+\+[header payload]\+ \texorpdfstring{$<$}{<}-\/ \doxylink{class_header_buffer}{Header\+Buffer} writes/\+reads this \+[2-\/byte len]\+\+[record payload]\+ \texorpdfstring{$<$}{<}-\/ data record 1 \+[2-\/byte len]\+\+[record payload]\+ \texorpdfstring{$<$}{<}-\/ data record 2 ...

Data record payload (packed binary):\+ \+[4-\/byte int zip\+Code]\+ \+[c-\/string place\+Name\textbackslash{}0]\+ \+[c-\/string state\textbackslash{}0]\+ \+[c-\/string county\textbackslash{}0]\+ \+[8-\/byte double latitude]\+ \+[8-\/byte double longitude]\+
